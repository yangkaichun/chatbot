1. 前言

隨著人工智能技術的飛速發展，醫療服務領域正經歷著數字化轉型的浪潮。亞東醫院作為台灣領先的醫療機構，致力於將創新技術應用於提升患者體驗和醫療服務效率。本計畫旨在設計並實施一套整合語音與文字互動的智能客服系統，為患者提供便捷、精準的醫療諮詢和掛號服務。

1.1 專案背景

當前醫院客服系統面臨多項挑戰：電話諮詢高峰期等待時間長、常規問題重複解答消耗人力資源、掛號選科需要專業指導等。智能語音文字客服系統將能有效解決這些問題，通過AI技術提供全天候、高效率、專業化的醫療諮詢和掛號推薦服務。

1.2 專案目標

本專案旨在打造一個集語音與文字互動於一體的亞東醫院智能客服系統，實現以下目標：

  • 建立基於亞東醫院專業知識的AI智能應答系統
  • 整合醫院官方資訊與ChatGPT，提供準確專業的諮詢服務
  • 根據患者描述的症狀，推薦合適的掛號科室和醫師
  • 與亞東醫院現有總機系統(IVR)無縫整合
  • 設計彈性化的後台管理，便於知識更新和系統優化
  • 實現系統模組化設計，支持快速部署到其他醫院

1.3 專案價值

患者價值

  • 提供24/7不間斷的醫療諮詢服務
  • 減少掛號等待時間與選科困惑
  • 多渠道交互方式，提升用戶體驗
  • 專業化的症狀分析與掛號建議

醫院價值

  • 釋放人工客服資源，專注處理複雜問題
  • 優化科室資源分配，提高就診精準度
  • 收集患者諮詢數據，優化醫院服務
  • 提升醫院品牌形象與技術先進性

計畫特色：

本系統除具備基本的AI客服功能外，特別增強了三大核心能力：
1. 專業的醫療症狀分析與科室推薦
2. 與醫院總機IVR系統的深度整合
3. 多租戶架構設計，支持跨醫院快速部署

2.2 核心組件

前端界面

響應式Web應用，支援手機瀏覽器訪問，提供語音與文字輸入功能，展示掛號推薦結果和醫師信息。

4. 系統架構關聯圖

以下展示系統各組件之間的數據流與交互關係，說明系統如何協同工作。

4.1 數據流程圖

圖4.1 智能客服系統數據流程圖

數據流程說明：

  1. 用戶通過網頁/APP/IVR提交文字或語音輸入
  2. 語音輸入經Azure Cognitive Services轉換為文字
  3. Dify平台的NLP引擎處理用戶輸入，理解意圖
  4. 系統根據意圖分類，路由至相應處理模塊（醫療諮詢/掛號推薦/其他服務）
  5. 處理模塊查詢知識庫和/或外部API獲取所需數據
  6. 生成回應，經過安全審核後發送給用戶
  7. 系統記錄互動數據，用於持續優化服務質量

4.3 掛號推薦邏輯流程

圖4.3 基於症狀的科室推薦邏輯流程

科室推薦邏輯說明

推薦系統基於多維度匹配原則，考量以下因素：

  1. 主症狀與疾病關聯強度
  2. 輔助症狀匹配程度
  3. 症狀持續時間與嚴重程度
  4. 人口統計特徵（年齡、性別等）相關性
  5. 歷史推薦成功率反饋

系統將綜合上述因素計算推薦分數，並按分數高低排序向用戶推薦最適合的科室。

6. AI介面

AI介面是系統的核心組件，基於Dify平台構建，提供自然語言理解和生成能力。

6.1 Dify平台配置

圖6.1 Dify平台應用配置界面

Dify平台主要配置：

  • 應用類型：對話式應用，支持多輪對話
  • 基礎模型：選擇適合醫療場景的大型語言模型
  • 提示詞模板：針對醫療諮詢和掛號推薦優化的專業提示詞
  • 知識庫設置：連接亞東醫院專業知識庫
  • API配置：設置與外部系統的連接參數

6.2 對話流程設計

圖6.2 AI對話流程設計

對話流程主要階段：

  1. 意圖識別：分析用戶輸入，確定主要意圖
  2. 信息提取：從用戶描述中識別關鍵信息（症狀、時間等）
  3. 知識檢索：從知識庫中檢索相關信息
  4. 回應生成：結合檢索結果和上下文，生成自然、專業的回應
  5. 後續引導：根據對話進展，提供相應的引導和建議

6.3 醫療專業性保障機制

確保醫療專業性的機制

  • 知識來源監控：所有回答基於亞東醫院提供的專業醫療知識
  • 專業審核流程：知識內容經醫療專家審核確認
  • 答案安全閾值：設置置信度閾值，低於閾值的問題引導至人工
  • 醫療免責聲明：清晰說明AI回答不替代專業醫療診斷
  • 危急情況識別：對嚴重症狀的描述，系統會建議立即就醫

6.4 語音處理能力

系統整合Azure Cognitive Services提供高質量的語音處理功能：

  • 語音識別(STT)：支持繁體中文及多種口音和方言識別
  • 語音合成(TTS)：自然流暢的語音輸出，支持調整語速和音調
  • 醫療術語優化：針對醫療專業術語的特殊處理
  • 噪音過濾：優化嘈雜環境下的語音識別率
  • 多平台支持：在Web、移動設備和電話系統上一致的語音體驗

7.3 系統安全策略

系統實施以下安全措施：

  • 網絡安全：使用Azure VNet進行網絡隔離，應用網關提供WAF保護
  • 數據安全：靜態和傳輸中的數據加密，敏感信息脫敏處理
  • 身份驗證：Azure AD集成，支持單點登錄和多因素認證
  • 授權控制：基於角色的細粒度權限控制
  • 安全監控：Azure Security Center持續監控，入侵檢測
  • 合規保障：符合醫療行業相關數據安全法規

7.4 擴展性與可用性設計

系統設計支持根據需求自動擴展：

  • 水平擴展：App Service可根據負載自動擴展實例數量
  • 垂直擴展：可根據需要調整計算資源規格
  • 區域冗餘：重要數據採用區域冗餘存儲
  • 備份策略：定期數據備份與災難恢復計劃
  • 高可用性：通過負載均衡和多實例部署確保可用性

系統知識庫整合多種來源的醫療信息，並通過彈性的後台管理實現快速更新和優化。

9.1 知識庫來源架構
